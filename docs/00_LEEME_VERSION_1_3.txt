EMMAGINA v1.3 ESTABLE

Objetivo de esta versión:
- Corregir carga de productos reales desde MongoDB/backend.
- Evitar que la tienda quede eternamente en "cargando" cuando los productos existen en la API.
- Preparar una base visual y técnica más limpia para seguir optimizando la experiencia tipo e-commerce profesional.

Cambios principales frontend:
1. API_BASE_URL apunta a https://emmagina-backend.onrender.com/api.
2. USE_STATIC_KSD_CATALOG queda en false para usar datos reales.
3. Se aumentó el tiempo de espera para Render y se agregaron reintentos.
4. Se aceptan más formas de respuesta del backend: productos, items, products y data.
5. Los productos personalizables también pueden aparecer en catálogo si están activos.
6. Si el campo publicarCatalogo queda ausente o inconsistente, la tienda muestra productos activos como respaldo.
7. Se ordenan imágenes de Cloudinary usando principal y orden cuando existan.
8. Se actualizó el cache busting de scripts/estilos a v1.3.0.

Cambios principales backend:
1. Versión actualizada a 1.3.0-emmagina-stable.
2. Se mantiene la ruta pública GET /api/productos.
3. Se mantienen alias temporales GET /api/producto y GET /api/catalogo.
4. Se agregó tolerancia de CORS para https://emmagina-frontend.onrender.com como origen de producción de Emmagina.

Cómo publicar:
1. Reemplazar el contenido del repositorio emmagina-frontend por esta carpeta.
2. git add .
3. git commit -m "Emmagina v1.3 estable"
4. git push
5. Esperar deploy del frontend en Render.
6. Repetir lo mismo con emmagina-backend.
7. Probar:
   - https://emmagina-backend.onrender.com/api/health
   - https://emmagina-backend.onrender.com/api/uploads/estado
   - https://emmagina-backend.onrender.com/api/productos
   - https://emmagina-frontend.onrender.com

Si los productos aparecen en /api/productos pero no en la tienda:
1. Abrir DevTools del navegador.
2. Ir a Console y revisar errores rojos.
3. Ir a Network, filtrar por productos y revisar si /api/productos responde 200.
4. Revisar que cada producto tenga activo=true.
5. Revisar que publicarCatalogo no esté explícitamente en false.
6. Revisar que el frontend haya cargado scripts con ?v=1.3.0.

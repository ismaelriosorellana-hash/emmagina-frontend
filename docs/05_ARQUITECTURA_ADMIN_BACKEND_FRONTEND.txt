ARQUITECTURA RECOMENDADA

Herramientas actuales:
- Frontend: HTML, CSS y JavaScript estático en Render Static Site.
- Backend: Node/Express en Render Web Service.
- Base de datos: MongoDB Atlas.
- Imágenes: Cloudinary.
- Repositorios: GitHub frontend y backend separados.
- Pagos futuros: Mercado Pago.
- Correo automático: Resend.
- Correo operativo: Zoho Mail.

Flujo actual:
Admin crea producto -> Backend guarda en MongoDB -> Frontend lee /api/productos -> Imagen viene desde Cloudinary -> Cliente compra o consulta.

Prioridad técnica:
1. Mantener frontend y backend separados.
2. Evitar editar MongoDB manualmente.
3. Usar panel admin para productos, categorías, banners y pedidos.
4. Mantener variables secretas solo en Render.
5. No subir .env a GitHub.

Módulos que debe tener el admin:
- Productos.
- Categorías.
- Banners.
- Pedidos.
- Crea tu escena.
- Inventario/materiales.
- Reportes.
- Configuración del sitio.
- Correos/notificaciones.

Sugerencia futura:
Cuando el sitio tenga más tráfico o se vuelva difícil mantener HTML/JS puro, evaluar migración controlada a Vite + React. No es necesario ahora; primero conviene estabilizar productos, admin, checkout y pedido personalizado.
